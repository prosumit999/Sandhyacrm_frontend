================================================================================
  SECTION 1 — OVERALL DESIGN PRINCIPLES
================================================================================

  THEME
  ─────
  Light theme ONLY — no dark mode. Never add dark: variants.
  color-scheme on root must stay "light".

  DESIGN REFERENCE
  ────────────────
  Style inspiration: Zoho CRM login screen
  Key traits: minimal, clean, professional, business-focused, no AI-generated feel.
  What that means:
    - No decorative gradients on form-side panels
    - No rounded pill buttons (use slightly-rounded rectangles)
    - Lots of whitespace — don't fill every pixel
    - Conservative color use — blue only for CTAs and focus rings
    - Content-first — no patterns, textures, or heavy illustrations on login

  COLORS
  ──────
  Primary blue (CTAs, focus, links) : #1a73e8
  Primary blue hover                 : #1557b0
  Primary blue disabled              : #93c5fd

  Page background (outer)            : #ececec
  Card background                    : #ffffff
  Right panel tint                   : #f8faff  (very slight blue tint on illustration panels)
  Sidebar (when built)               : TBD — keep consistent with this system

  Text primary                       : #111827
  Text secondary / muted             : #6b7280
  Text placeholder                   : #9ca3af
  Text heading (strong)              : #1e293b

  Border default                     : #d1d5db
  Border focus                       : #1a73e8
  Divider                            : #e5e7eb

  Error text                         : #dc2626
  Error background                   : #fef2f2
  Error border                       : #fecaca

  Success text                       : #16a34a
  Success background                 : #f0fdf4

  Warning (soft)                     : #d97706

  Badge — Active / Live              : green (#16a34a bg light)
  Badge — Expired / Broken           : red   (#dc2626 bg light)
  Badge — Pending                    : amber (#d97706 bg light)
  Badge — Draft                      : gray  (#6b7280 bg light)

  TYPOGRAPHY
  ──────────
  Font family  : 'Inter', system-ui, -apple-system, 'Segoe UI', sans-serif
  Base size    : 14px (body), 13-13.5px (form fields, labels), 12px (meta/footer)
  Headings     : 27-28px (page titles), 16-18px (section headings), 13.5px (card titles)
  Weight       : 700 (h1/h2), 600 (button text, labels), 500 (subheadings), 400 (body)
  Letter spacing: -0.4px on large headings, +0.1px on button text, +0.8px on logo text

  SPACING (use these consistently — do not eyeball)
  ──────────────────────────────────────────────────
  Card padding               : 48-52px (desktop panels)
  Logo → heading gap         : 44px margin-bottom on logo block
  Heading → form gap         : 26px margin-top on form
  Field vertical gap         : 12-14px between stacked inputs
  Button margin-top          : 20px from last field
  Section internal padding   : 32px
  Page outer padding         : 24px (mobile) / 48px (desktop)

  SHADOWS
  ───────
  Card (main)     : 0 2px 20px rgba(0,0,0,0.09)
  Dropdown/popup  : 0 4px 16px rgba(0,0,0,0.12)
  Hover state     : 0 2px 8px rgba(0,0,0,0.08)
  No shadow       : inputs, text areas, badges

  BORDER RADIUS
  ─────────────
  Cards / modals     : 12px
  Input fields       : 6px
  Buttons            : 6px
  Logo icon squares  : 8px
  Small badges       : 4px
  Avatars / circles  : 50%

  ANIMATION POLICY
  ────────────────
  Library: Framer Motion — use SPARINGLY.
  Allowed:
    - Page/card mount: single fade (opacity 0→1) + slight slide (y: 14→0), duration 0.3s ease-out
    - Error banner: fade + slide up (y: -6→0)
    - Modal open: fade + scale (scale 0.97→1)
  NOT allowed:
    - Hover animations on data cells or nav items (use CSS transition: color 0.15s only)
    - Staggered list animations
    - Looping or continuous animations (except loading spinners)
    - Page exit animations (only enter)
  Loading spinner: inline SVG, CSS animation loginSpin, 0.75s linear infinite

================================================================================
  SECTION 4 — AUTH FLOW (critical — read before touching auth)
================================================================================

  Backend uses httpOnly cookies — NOT localStorage tokens.
  Two cookies set on login:
    logintoken    : access token, expires 15 min
    refreshToken  : refresh token, expires 7 days

  Frontend rules:
  1. axiosInstance has withCredentials: true — browser sends cookies on every request
  2. On 401: interceptor calls POST /auth/refresh (re-issues logintoken cookie), then retries
  3. If /auth/refresh also returns 401: redirect to /login
  4. On app startup (main.jsx): dispatch initializeAuth() → GET /auth/me
     - If cookie still valid: populates user in Redux, isAuthenticated = true
     - If expired/missing: isAuthenticated = false, stays on /login
  5. Redux auth state shape:
       { user, isAuthenticated, initializing, loading, error }
     - initializing: true until initializeAuth() settles (prevents flash redirect on page refresh)
  6. NEVER store tokens in localStorage or sessionStorage
  7. logout: dispatch logoutUser() → POST /auth/logout → backend clears cookies → Redux reset

  Login response body: { success, message, user: { id, name, email, role } }
  Token is NOT in the response body — it's in the httpOnly cookie.

  IMPORTANT: Tailwind v4 has no tailwind.config.js — use CSS source declarations.
  IMPORTANT: Backend runs on port 5000. Frontend runs on port 5173 (Vite default).
